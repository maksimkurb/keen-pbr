# Конфигурация

keen-pbr читает свои настройки из одного JSON-файла конфигурации.

Расположение файла конфигурации:

- Keenetic / NetCraze: `/opt/etc/keen-pbr/config.json`
- OpenWrt: `/etc/keen-pbr/config.json`
- Debian: `/etc/keen-pbr/config.json`

Если вы установили полный пакет, вы обычно можете выполнить первую настройку в веб-интерфейсе и вернуться к этому разделу позже. Большинству пользователей нужны только эти четыре части конфигурации:

- [Outbounds](outbounds/) — куда должен идти трафик
- [Списки](lists/) — сайты или диапазоны IP для сопоставления
- [Правила маршрутизации](route-rules/) — какие списки через какой outbound
- [DNS](dns/) — какой DNS-сервер использовать для этих списков

## Практический пример

Этот пример маршрутизирует `google.com` через `vpn` и оставляет всё остальное на `wan`:

```json
{
  "outbounds": [
    {
      "type": "interface",
      "tag": "vpn",
      "interface": "tun0",
      "gateway": "10.8.0.1"
    },
    {
      "type": "interface",
      "tag": "wan",
      "interface": "eth0",
      "gateway": "192.168.1.1"
    }
  ],
  "lists": {
    "my_sites": {
      "domains": ["google.com"]
    }
  },
  "dns": {
    "system_resolver": {
      "type": "dnsmasq-nftset",
      "address": "127.0.0.1"
    },
    "servers": [
      {
        "tag": "vpn_dns",
        "address": "10.8.0.1",
        "detour": "vpn"
      },
      {
        "tag": "default_dns",
        "address": "1.1.1.1"
      }
    ],
    "rules": [
      {
        "list": ["my_sites"],
        "server": "vpn_dns"
      }
    ],
    "fallback": ["default_dns"]
  },
  "route": {
    "rules": [
      {
        "list": ["my_sites"],
        "outbound": "vpn"
      }
    ]
  }
}
```

## Базовая конфигурация

- [Outbounds](outbounds/) — выберите VPN и обычное интернет-соединение
- [Списки](lists/) — определите сайты, домены или диапазоны IP для сопоставления
- [Правила маршрутизации](route-rules/) — соедините каждый список с outbound
- [DNS](dns/) — убедитесь, что домены сопоставления разрешаются через правильный DNS-сервер

## Расширенная конфигурация

Эти настройки необязательны для большинства пользователей:

- [Расширенные](advanced/) — API, пути сервиса, автоматическое обновление списков и низкоуровневые параметры маршрутизации

{{< callout type="info" >}}
Имена списков, теги outbound и теги DNS-серверов должны соответствовать одному соглашению: `^[a-z][a-z0-9_]*$` и должны быть не более 24 символов.
{{< /callout >}}
