# Обновление с 2.x

Используйте это руководство, если у вас уже работает keen-pbr 2.x на Keenetic / NetCraze и вы хотите перейти на текущий пакет и формат конфигурации.

## 1. Замените репозиторий

Откройте страницу инструкций репозитория, выберите **Keenetic / NetCraze** в селекторе ОС слева, выберите архитектуру вашего роутера и выполните сгенерированные команды, чтобы заменить старый репозиторий keen-pbr 2.x на текущий:

{{< hextra/hero-button text="Репозиторий keen-pbr" link="https://repo.keen-pbr.fyi/repository/stable" >}}

Используйте команды именно в том виде, как они показаны на странице репозитория для Keenetic / NetCraze.

{{< callout type="info" >}}
Перед заменой пакетов сохраните резервную копию текущей конфигурации 2.x, чтобы при необходимости можно было сравнить её с результатом конвертации.
{{< /callout >}}

## 2. Установите keen-pbr

После замены репозитория установите текущий пакет.

### Keenetic / NetCraze

```bash {filename="bash"}
opkg install keen-pbr

# или если нужна версия без API и без веб-интерфейса
# opkg install keen-pbr-headless
```

Путь к конфигурации:

```text
/opt/etc/keen-pbr/config.json
```

## 3. Сконвертируйте конфигурацию в новый формат

Откройте инструмент для [конвертрации конфига keen-pbr](/converter/?lang=ru)

Затем:

1. Скопируйте содержимое `/opt/etc/keen-pbr/keen-pbr.conf` в поле ввода конвертера слева.
2. Проверьте сгенерированный конфиг.
3. Замените содержимое `/opt/etc/keen-pbr/config.json` на результат конвертации.

{{< callout type="info" >}}
Заменяйте конфигурационный файл только после проверки содержимого.
{{< /callout >}}

## 4. Перезапустите keen-pbr

После сохранения конфига перезапустите сервис.

```bash {filename="bash"}
/opt/etc/init.d/S80keen-pbr restart
```

После перезапуска откройте [Быстрый старт](../quick-start/), если хотите проверить сервис через веб-интерфейс или продолжить настройку через JSON / CLI.
